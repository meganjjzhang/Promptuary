# Obsidian 批注插件调研报告

> 日期：2026-06-10 | 需求场景：Obsidian 批阅文档时，零散修改想法难记录、难一次性喂给 AI 修改

---

## 一、需求拆解

核心痛点是**批注 → 汇总 → AI 执行**这条链路断了：
1. **记录难**：阅读时零散想法没有低摩擦的记录方式
2. **喂 AI 难**：攒了一堆修改点，无法一次性结构化地交给 AI 执行

必须满足：
- 选中即批注（低摩擦）
- 不污染原文
- 侧边栏汇总
- 批注汇总 → AI Prompt → 批量执行 → Diff 预览

---

## 二、三插件深度对比

### 1. Sidebar Highlights（⭐137，GPL-3.0）

**做什么**：高亮 + 评论 + 任务管理的侧边栏聚合工具

**核心能力**：
- 3 种评论语法：`==高亮==[^1]` 脚注评论 / `==高亮==^[内联评论]` / `%%独立评论%%`
- 侧边栏 4 个 Tab：当前笔记 / 全部笔记 / 集合 / 任务
- 5 色高亮（金/红/青/蓝/绿）
- 集合系统：跨文件组织高亮
- 强大搜索：支持 `AND/OR`、`#tag`、`@collection`、`-排除`
- 任务管理：自然语言日期、按截止日期分组
- 国际化：完整简体中文支持
- 24 个版本迭代，较为成熟

**致命缺陷**：
- ❌ **改原文**：高亮和评论直接写入 Markdown，污染文档
- ❌ 无 AI 能力
- ❌ 无批量修改执行

**可借鉴**：
- 分类/搜索/集合体系设计精良
- 交互模型（右键高亮、侧边栏 Tab）可直接参考

---

### 2. HiNote（⭐241，MIT，Beta）

**做什么**：高亮文本管理 + AI 注释 + 导出的学习工具

**核心能力**：
- 自动提取高亮（`==`、`<mark>`、`<span>`、自定义正则）
- 为高亮添加注释（卡片式 UI）
- AI 辅助注释：支持 OpenAI/Gemini/Anthropic/Deepseek/Ollama
- 导出为图片（知识卡片）或 Markdown（Callout 格式 + Block ID）
- Pro 版：FSRS 间隔重复闪卡

**致命缺陷**：
- ❌ **改原文**：高亮直接在原文标记
- ❌ 无批量执行修改（AI 只做注释，不改文档）
- ❌ 无搜索/过滤功能
- ❌ Beta 阶段，数据安全需谨慎

**可借鉴**：
- AI 注释的工作流设计（配置 Provider → 自定义 Prompt → 触发 AI）
- 导出为笔记的 Block ID 回链设计

---

### 3. Axl Light（⭐1，MIT，v1.5）

**做什么**：轻量级标注插件，高亮 + 便签，数据与原文分离

**核心能力**：
- 6 色高亮 + 便签注释
- **sidecar 存储模式**：所有注释数据存储在 `.obsidian-annotations/` 隐藏文件夹，原文完全不动
- 支持文件重命名追踪
- 导出为 Markdown
- 支持 PDF 标注

**致命缺陷**：
- ❌ 极不成熟（1 star，18 commits）
- ❌ 无跨文件汇总视图
- ❌ 无 AI 能力
- ❌ 无批量执行
- ❌ 仅支持 Windows

**可借鉴**：
- **sidecar 存储架构**：注释与原文完全分离，这是最关键的架构决策
- 便签在右侧专用区域显示的交互设计

## 四、结论

### 三个插件都无法直接满足需求

共同缺失的关键链路：**批注 → AI Prompt 生成 → 批量执行 → Diff 预览**

### 整合方案：基于 multiaiedit 自研

```
┌─────────────────────────────────────────────────┐
│              multiaiedit 插件架构                │
├─────────────────────────────────────────────────┤
│                                                 │
│  ① 批注输入层                                   │
│     选中文本 → 右键/快捷键 → 批注面板           │
│     参考：Sidebar Highlights 交互模型            │
│                                                 │
│  ② 存储层（借鉴 Axl Light）                     │
│     sidecar JSON 文件，原文零污染                │
│     支持批注分类：修改/疑问/删除/补充            │
│                                                 │
│  ③ 汇总视图层（借鉴 Sidebar Highlights）         │
│     侧边栏：当前文档 / 全库 / 按分类过滤        │
│     搜索、集合、颜色标记                         │
│                                                 │
│  ④ AI 协作层（借鉴 HiNote + 自研）              │
│     AI 生成批注建议（HiNote 思路）               │
│     一键汇总所有批注为结构化 Prompt              │
│     调用 AI 批量执行修改                         │
│                                                 │
│  ⑤ 执行确认层（自研核心）                       │
│     Diff 预览：逐条展示修改前后对比             │
│     逐条确认 / 全部接受 / 拒绝                   │
│     执行后自动清理已处理批注                     │
│                                                 │
└─────────────────────────────────────────────────┘
```

### MVP 优先级

| 阶段 | 功能 | 优先级 |
|------|------|--------|
| P0 | 选中 → 批注（sidecar 存储） | 必须 |
| P0 | 侧边栏批注列表 | 必须 |
| P0 | 一键生成 AI Prompt + 批量执行 + Diff 确认 | 必须 |
| P1 | 批注分类/颜色 | 高 |
| P1 | 跨文件批注看板 | 高 |
| P2 | 搜索/过滤 | 中 |
| P2 | AI 辅助批注建议 | 中 |
| P3 | 导出为 Markdown | 低 |

## 五、短期替代方案（插件开发完成前）

### 方案 A：内联标记法（立即可用）

在文档中用 `%%AJ: 修改指令%%` 插入批注：
- 阅读模式自动隐藏
- 编辑模式可见
- 直接把文档丢给 AI：*"找出所有 %%AJ: ...%% 标记，按标记内容修改文档，修改完成后删除标记"*

### 方案 B：批注收集文件（配合 A 更强）

建一个 `批注汇总.md`，用双链 + checkbox 跟踪：
```markdown
# 批注汇总
## [[产品定位文档]]
- [ ] 第3行 "家长" → "新生代妈妈"
- [ ] 第5行 卖点前置到标题
- [x] 第12行 错别字已修正
```

### 方案 A+B 组合 = 今天就能用的完整流程
